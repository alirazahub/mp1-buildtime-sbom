# MP1: Build-time SBOM Comparison

## Software Supply Chain Security – Mini Project 1

---

# 1. Project Overview

This project explores the differences between Software Bill of Materials (SBOM) generation approaches at different stages of the software supply chain.

The main goal is to compare:

- Source-level SBOMs
- Container/Image-level SBOMs
- SBOMs generated by different tools

The project investigates how dependency visibility changes between:
- application source code
- build-time artifacts
- final container runtime environments

This helps understand:
- missing dependencies
- unexpected runtime components
- build/runtime mismatches
- dependency granularity differences

---

# 2. Project Objective

The purpose of this mini project is to evaluate the effectiveness and completeness of build-time SBOM generation approaches.

Specifically, the project compares SBOMs generated:
- directly from application source directories
- from built Docker container images

The project also compares two popular SBOM generation tools:
- Syft
- Trivy

The expected outcome is to identify:
- dependency differences
- runtime-only packages
- transitive dependencies
- OS-level packages
- limitations of source-only dependency scanning

# 7. Key Findings

## Node.js

- Syft discovered more source dependencies than Trivy.
- Runtime images contained Alpine Linux packages and Node.js runtime packages.
- Source scans missed some runtime dependencies.

## Python

- Very large difference between source and image SBOMs.
- Container images introduced Debian runtime packages and system libraries.
- Build-time SBOMs exposed hidden runtime dependencies.

## Go

- Smaller difference because Go compiles dependencies into binaries.
- Image scans still revealed additional runtime packages.

---

# 8. Major Observations

- Source-level SBOMs mainly capture application dependencies.
- Image-level SBOMs capture runtime and operating system dependencies.
- Syft performs deeper recursive dependency analysis.
- Trivy provides cleaner manifest-oriented dependency discovery.
- Build-time SBOMs provide better visibility into deployed software artifacts.

---

# 9. Conclusion

This project demonstrated that build-time and image-level SBOM generation provide significantly more complete visibility into deployed software systems compared to source-only SBOM generation.

Image-level SBOMs reveal:
- runtime libraries
- operating system packages
- interpreters
- hidden transitive dependencies

These components are often invisible in source-only scans.

Therefore, build-time SBOMs improve:
- supply chain visibility
- deployment transparency
- dependency tracking
- security analysis
